    • Tree Window Functionality
      • "Sync from Text Window" Button
        Syncs over data from the Text Edit box on the right. For performance reasons it is not prudent to update the tree on each key stroke in the text edit box. So instead we mark a status Led telling you a sync is needed, and then you can decide when you want do a sync.

      • "Tree Synced" LED
        Indicates if the tree needs to be synched from the Text Edit box on the right. If Green, they are synced. If Red they are not synced.

      • "Sort By Gal, Sys, Name" Button
        Sorts all top level Base Nodes by Galaxy, then by System within Galaxy,and then alphbetically by Name within the System. This function is really what motivated the entire application. In one click you have your bases organized in some way that makes sense.

      • "Background Processing" LED
        Indicates if there is background processing happening. This app is intended to be able to work with very large files, but certain operations can cause lag and slowness in the app until they complete. (This was true especially early in development; at this point I have revised most operations observed to cause lag, however...) If this LED turns yellow YOU MAY STILL attempt further app use, but if you find you are not getting an immediate response to new inputs you may need to wait until the LED turns green again. When the LED is green the app is in an idle state with little processing happening. (For techies: it's counting the number of child threads for the app and thresholds determined via observation)

      • Move Freight to Top on Sort checkbox
        When check, your freighter base will be sorted to the top of the list regardless of any other sort criteria.

      • Tree Drag and Drop Functionality
        On the Base and Starship Processing Tabs, Tree data nodes may be dragged and dropped to manually reorder within the tree. Tree view changed are automatically synced to the text window. Child nodes will be prevented from moving unless they have the same data type in their parent nodes. Dragging upward amoungst direct siblings will drop the dragged node above above the targetted drop target. Dragging downward amoungst direct siblings will drop the dragged node below the targetted drop target. If you drop to a target under a different parent, the draggd node will always drop below the drop target. The primary tested purpose for this is to allow manual reordering of the highest level base tree nodes.

    • Text Area Window Functionality
      • "Import from Clipboard" Button"
        This will REPLACE the full text in the text area window, with the text from your system clipboard as if you had done a select all, and paste. This is the easiest way to enter your current base data into this application. All views are updated with the new data.

      • "Export to Clipboard" Button"
        This will COPY the full text from your text area window, into the system clipboard as if you had done a select all, and copy. This is the easiest way to grab all the data from this application, to be saved to the live game via Goatfungus or NomNom.

      • "Pretty Print" Button"
        Reformats the bases Json in the text window in a more organized way.

      • Text Area Right Click Menu
        In this window the functionality will be pretty standard to any text editor. Of note, there is a search text feature.

      • "Upgrade Starships" Button
        The functionality of this button will be to Upgrade all of your Starships as pasted into the text window, with one click. Read on to understand process and setup.

        • General Functionality
          The way it works is, for each Starship, your Starship Name and Seed are saved off. The Starship Seed is a hex data value that is used by the game to specify all aesthetic aspects of your Starship. Build parts, color, physical configuration. We then replace each Starship's complete data with upgraded reference Starship data for the same type (Fighter, Sentinel, Hauler, etc) of Starship as the original. We then overwrite the Name and Seed of the reference data with your orginal ship's data. In this way you end up with what is basically the same ship, but with all the upgrades applied to it. (Please read on...)

        • Reference Ship Data
          For Each ship type of Explorer, Fighter, Hauler, Living, Royal, Sentinel, Shuttle, Solar (other ship types not yet tested, YMMV), there is a reference file that lives in the same directory as the program files, called respectfully: reference_explorer.json, reference_fighter.json, reference_hauler.json, reference_living.json, reference_royal.json, reference_sentinel.json, reference_shuttle.json, and reference_Solar.json. These come with the data I use to upgrade my own ships, but they each contain the Json for a standard starship as you can grab from the other save editors. If you want to apply your own upgrades you would just copy the Json data for a starship you want to use, and paste it into the appropriate reference file for the ship type. From that point on your data would be used for that Starship type.

        • Actual Button Usage
          When you hit the button you will be presented a dialog box allowing you to pick the starships you wish to upgrade. Only those startships indicated will be upgraded. Just check off those you want to upgrade and hit enter. The results will be propegated to the Tree and Text windows.

          ***IMPORTANT***: Do NOT attempt to upgrade custom built or expedition ships (Iron Vulture, Starborn Runner, etc)!!! I'll say it again: DO NOT ATTEMPT TO UPGRADE CUSTOM BUILT OR EXPEDITION SHIPS!!! These ships use some additonal mechanism to determine physical characteristics of the ship beyond just a seed value, and honestly I'm not sure what that is. But I do know that if you use this application to attempt to upgrade a custom built or expedition ship, you will end up with a standard hunk of crap looking ship as a result!

  • Teleport Endpoints Tab
    • Explanation
      Within an NMS save file there is a section called "Teleport Endpoints" that stores some number of previous places you have teleported. It can be up into the hundreds of endpoints. The main purpose of this tab section is to allow you to set your current player location to the approximate location of such an endpoint (see the right click menus). I use it for two purposes. 1) to attempt to find some location from my past that I thought was lost. 2) Just as another fun way to explore. So many times I am in some area and don't really do it justice in exploration before I move on, and I never threw a base there, but if it's on this list obviously *something* caused me to want to go there. I will use this feature to give me an option to re-explorer places on the theory that at times maybe I didn't give a place enough time.

      If you use this feature to teleport to a base that has since been removed and the portal is no longer there, it seems that the game places you in the space station for that system. Testing has shown it will at times put you right at the location where the advertised portal was, but I would advise that you should view this more like repeating exploration of a previous *area* that you had been to rather than expecting it will put you dead smack on a previous portal location every time.

  • Tree Right Click Menu
    • All Tabs - Common to all Tree Nodes
      • Highlight Text in Box
        Will highlight and grab focus on the data source line corrisponding to right clicked tree node, in the text window on the right hand side.

      • Toggle Expansion 2 Levels

        This will toggle node expansion. If the the node starts out closed, it will expand two levels deep from the right clicked node. If the node was already opened, it's just going to close the right clicked node.

      • Delete Node
        Deletes the right clicked node.

    • 'Base Processing' Tab - ***Specific to top level Base Nodes***
      • Back Up a Base
        Allows saving off a single base. The main thought for this feature within this tool is really to be able to easily move bases from one save to another. It's obviously not going to complete with other save file editors for organizational feature richness in this area.

    • 'Base Processing' Tab - ***Specific to the very top most Array Node ONLY***
      • Restore a Base
        Will allow you to pick a previously backed up base file to create and insert a new Base node at the top of the tree.

    • 'Inventory Processing' Tab - Specific to the top level Inventory Processing tree nodes
      • Backup this Inventory
        Saves off all items in this inventory to a file. Be aware that on importing later via 'Replace Parts from Backup', all parts in the target are replaced by the saved off parts.

        The functionality here is meant to allow you to keep backups sets of default inventory items to be placed in any of the inventory containers on demand.

      • Replace Items from Backup
        This will clear existing parts and replace them with the items from the saved file.

        Different from backing and restoring up a base on the Base Processing tab, the Replace/Restore functionality here does NOT create a new storage container. The logic for this feature essentially copies inventory items from the saved file, into the selected existing target inventory (that is, the inventory storage that you right clicked on to start this process) replacing whatever was there originally.
